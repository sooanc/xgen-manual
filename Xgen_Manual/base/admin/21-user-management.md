# 사용자 관리

본 챕터는 시스템 사용자 계정의 **승인·수정·비활성화** 절차를 다룹니다. **신규 사용자 추가는 본인이 직접 가입 → 활성 대기 큐에 등록 → SuperUser 가 활성 승인** 의 흐름으로 처리됩니다. *관리자가 직접 사용자 정보를 입력해 등록하는 "+ 사용자 추가" 버튼은 현재 stg 화면에 노출되지 않습니다.*

## 사용자 목록 { #user-list }

좌측 메뉴 **관리 설정 → 사용자 / 접근제어 → 사용자 관리**를 선택합니다.

![사용자 관리 화면 — 상단의 *전체 사용자 / 승인 대기* 통계 카드, *활성 대기 사용자* 큐, 검색·페이지네이션이 있는 전체 사용자 테이블이 한 화면에 노출](images/admin-users.png)

화면은 4개 영역으로 구성됩니다.

| 영역 | 표시 정보 |
|---|---|
| 상단 통계 | **전체 사용자** 수, **승인 대기** 건수 |
| 검색 입력 | 이메일·사용자명·이름으로 즉시 필터 |
| 활성 대기 사용자 큐 | 자체 가입한 신규 신청자 — 행 우측 **활성 / 삭제** 버튼으로 즉시 처리 |
| 전체 사용자 테이블 | 전체 계정 목록 — 행 우측 **편집 / 삭제** 버튼 |

전체 사용자 테이블의 컬럼은 다음과 같습니다. 컬럼 헤더 클릭으로 정렬 토글 가능.

| 컬럼 | 설명 |
|---|---|
| ID | 시스템 내부 식별자 |
| 사용자명 | 로그인 ID (자체 가입 시 입력한 영문 ID 또는 표시 이름) |
| 이메일 | 알림·비밀번호 재설정 수신 주소 |
| 상태 | 활성 / 비활성 / 대기중 |
| 권한(역할) | `superuser` / `standard` 등급 구분 + 부여된 역할 라벨 (예: *시스템 관리자(DJ)*, *에이전트개발자(DJ)*) |
| 마지막 로그인 | 최근 접속 시각 |
| 마지막 로그인 IP | 접속 IP |
| 등록일 | 가입 또는 생성 일시 |
| 액션 | **편집** / **삭제** 버튼 |

## 신규 사용자 추가 방식 { #user-add }

신규 사용자 계정은 조직의 운영 정책에 따라 아래 두 가지 방식으로 추가할 수 있습니다.

- **자체 가입(Self-signup) 후 관리자 승인 방식**
- **사내 SSO(Single Sign-On) 연동 방식**

본 절에서는 *자체 가입(Self-signup)* 환경을 기준으로 사용자 등록 및 승인 절차를 설명합니다.

### 신규 사용자 추가 — 자체 가입 후 관리자 승인

신규 사용자는 회원가입 페이지에서 직접 가입 신청을 진행하며, 가입 완료 후 **활성 대기** 상태로 등록됩니다. 이후 SuperUser 또는 사용자 관리 권한을 가진 관리자가 승인해야 로그인이 가능합니다.

#### 1단계 — 사용자 회원가입

신규 사용자는 아래 절차에 따라 직접 가입을 진행합니다.

1. 솔루션 로그인 페이지(`https://<도메인>/login`) 하단의 **회원가입** 링크를 선택합니다.
2. 회원가입 페이지(`/signup`) 에서 아래 정보를 입력합니다.
    - 이메일
    - 비밀번호
    - 표시 이름(Display Name)
3. **가입 신청** 을 완료합니다.

가입 신청 이후 계정은 *활성 대기* 상태로 등록되며, 관리자 승인 전까지는 로그인할 수 없습니다.

#### 2단계 — 관리자 승인

SuperUser 또는 사용자 관리 권한을 가진 관리자는 *사용자 관리* 화면에서 가입 신청 내역을 검토하고 계정을 승인합니다.

**사용자 승인 절차**

1. **관리 설정 → 사용자 / 접근제어 → 사용자 관리** 메뉴로 이동합니다.
2. 상단 통계 카드에서 **승인 대기** 사용자 수를 확인합니다.
3. *활성 대기 사용자* 목록에서 신청자 정보를 확인합니다.

    확인 가능한 정보:

    - ID
    - 사용자명
    - 이메일
    - 상태(대기중)
    - 등록일시

4. 신청 정보를 검토한 후 계정을 처리합니다.

    - **활성 처리**
        - **활성** 버튼 선택
        - 계정 상태가 *활성* 상태로 변경됨
        - 사용자가 즉시 로그인 가능
    - **삭제 처리**
        - **삭제** 버튼 선택
        - 가입 신청 정보 삭제
        - 사용자는 다시 회원가입을 진행해야 함

#### 참고 사항

- 자체 가입 기능 사용 여부는 조직 운영 정책에 따라 다를 수 있습니다.
- SSO 환경에서는 별도의 회원가입 절차 없이 사내 계정 인증을 통해 자동 로그인될 수 있습니다.
- 승인 및 계정 상태 변경 이력은 감사 로그(Audit Log) 에 기록될 수 있습니다.

!!! note "SuperUser 권한 필요"
    활성 대기 큐 승인·거부, 사용자 권한 등급 변경, 비밀번호 재설정은 모두 **SuperUser** 또는 동등한 사용자 관리 권한자만 수행 가능합니다. 일반 사용자(Standard User) 에게는 본 화면 자체가 노출되지 않습니다.

## 비밀번호 재설정

관리자가 다른 사용자의 비밀번호를 직접 재설정할 수 있습니다.

1. 사용자 목록에서 대상 사용자의 **편집** 클릭
2. 모달 하단 **비밀번호 변경** 섹션 펼치기
3. 새 비밀번호 입력 → **저장**

> **권장:** 비밀번호 재설정 후 사용자에게 별도 채널로 새 비밀번호를 전달하고, 첫 로그인 시 변경하도록 안내합니다.

## 사용자 비활성화

사용자를 시스템에서 제거하지 않고 일시적으로 차단할 때 사용합니다.

1. 사용자 목록에서 대상의 **편집** 클릭
2. 상태를 **비활성**으로 변경
3. **저장**

비활성 사용자는 로그인할 수 없으나, 그가 만든 에이전트플로우·컬렉션·채팅 이력은 그대로 보존됩니다.

!!! warning "삭제 vs 비활성화"
    가능한 한 **비활성화**를 사용하세요. **삭제**는 사용자 데이터까지 영구 제거되어 복구가 불가합니다. 퇴사·전배 같은 경우는 비활성화가 적절합니다.

## 권한 등급 변경

권한 등급은 2단계입니다. 사용자 편집 모달의 **사용자 유형** 셀렉트에서 `Standard` ↔ `Superuser` 를 전환해 변경합니다.

| 변경 방향 | 필요 권한 |
|---|---|
| Standard User → SuperUser (승격) | SuperUser |
| SuperUser → Standard User (강등) | SuperUser |

모든 권한 등급 변경은 감사 로그에 기록됩니다. 본인이 마지막 SuperUser 인 경우 자기 자신을 강등하면 관리 영역 접근 권한자가 사라지므로, 강등 전 다른 SuperUser 가 1명 이상 남아 있는지 확인하세요.

## 운영 권장사항

- **비밀번호 정책 통일** — 조직 비밀번호 정책이 있으면 사용자 추가 시 적용. 최소 길이·복잡도·만료 주기는 시스템 설정에서 일괄 관리.
- **휴면 계정 정리** — 마지막 로그인이 90일 이상 지난 계정은 분기별로 비활성화 검토.
- **공용 계정 금지** — 한 계정을 여러 사람이 쓰면 감사 로그 추적이 무의미해집니다. 1인 1계정 원칙.

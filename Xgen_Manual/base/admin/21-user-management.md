# 사용자 관리

본 챕터는 시스템 사용자 계정의 추가·승인·수정·비활성화 절차를 다룹니다.

## 사용자 목록

좌측 메뉴 **관리 설정 → 사용자 / 접근제어 → 사용자 관리**를 선택합니다.

![사용자 관리 목록 — 검색창, 상태 필터, 사용자 추가 버튼과 사용자별 정보 컬럼](images/admin-users.png)

각 사용자에 대해 다음 정보가 표시됩니다.

| 컬럼 | 설명 |
|---|---|
| 사용자명 | 로그인 ID |
| 이름 | 표시 이름 |
| 이메일 | 알림·비밀번호 재설정 수신 주소 |
| 역할 | 부여된 역할 (없을 수 있음) |
| 권한 등급 | 일반 사용자 / 관리자 / 최고 관리자 |
| 상태 | 활성 / 비활성 / 활성 대기 |
| 마지막 로그인 | 최근 접속 시각 |

상단 검색창에 사용자명·이메일을 입력해 빠르게 찾을 수 있고, 상태 필터로 "활성 대기"만 따로 볼 수 있습니다.

## 사용자 추가

1. 사용자 목록 우상단 **+ 사용자 추가** 버튼 클릭
2. 다음 항목 입력
    - 사용자명 (영문, 중복 불가)
    - 이름
    - 이메일
    - 초기 비밀번호 (사용자가 첫 로그인 후 변경 권장)
    - 권한 등급 (기본: 일반 사용자)
3. **저장** 클릭

!!! note "사용자 추가 모달 캡처는 별도 갱신 예정"
    "+ 사용자 추가" 클릭 시 노출되는 모달(사용자명·이메일·권한 등급 입력)의 캡처는 다음 회차에 보강 예정입니다.

!!! note "관리자 권한 필요"
    사용자 추가 및 권한 등급 변경은 **관리자(Administrator)** 이상의 권한이 필요합니다. 최고 관리자(Superuser) 부여는 최고 관리자만 가능합니다.

## 활성 대기 사용자 승인

자체 가입 모드가 활성화된 환경에서는 신규 가입 신청자가 **활성 대기** 상태로 들어옵니다. 관리자 검토 후 승인해야 로그인이 가능합니다.

1. 사용자 목록 상단 **활성 대기** 영역에서 신청자 확인
2. 신청 정보(이름·이메일·소속) 검토
3. **승인** 버튼 클릭 → 활성 사용자로 전환
4. 거절 시 **삭제** 클릭

## 비밀번호 재설정

관리자가 다른 사용자의 비밀번호를 직접 재설정할 수 있습니다.

1. 사용자 목록에서 대상 사용자의 **편집** 클릭
2. 모달 하단 **비밀번호 변경** 섹션 펼치기
3. 새 비밀번호 입력 → **저장**

> **권장:** 비밀번호 재설정 후 사용자에게 별도 채널로 새 비밀번호를 전달하고, 첫 로그인 시 변경하도록 안내합니다.

## 사용자 비활성화

사용자를 시스템에서 제거하지 않고 일시적으로 차단할 때 사용합니다.

1. 사용자 목록에서 대상의 **편집** 클릭
2. 상태를 **비활성**으로 변경
3. **저장**

비활성 사용자는 로그인할 수 없으나, 그가 만든 에이전트플로우·컬렉션·채팅 이력은 그대로 보존됩니다.

!!! warning "삭제 vs 비활성화"
    가능한 한 **비활성화**를 사용하세요. **삭제**는 사용자 데이터까지 영구 제거되어 복구가 불가합니다. 퇴사·전배 같은 경우는 비활성화가 적절합니다.

## 권한 등급 변경

| 변경 방향 | 필요 권한 |
|---|---|
| 일반 사용자 → 관리자 | 관리자 또는 최고 관리자 |
| 관리자 → 최고 관리자 | 최고 관리자만 |
| 모든 등급 → 일반 사용자 (강등) | 관리자 이상 |

권한 등급 변경은 모두 감사 로그에 기록됩니다.

## 운영 권장사항

- **비밀번호 정책 통일** — 조직 비밀번호 정책이 있으면 사용자 추가 시 적용. 최소 길이·복잡도·만료 주기는 시스템 설정에서 일괄 관리.
- **휴면 계정 정리** — 마지막 로그인이 90일 이상 지난 계정은 분기별로 비활성화 검토.
- **공용 계정 금지** — 한 계정을 여러 사람이 쓰면 감사 로그 추적이 무의미해집니다. 1인 1계정 원칙.

# 관리자 콘솔 개요

본 챕터는 시스템 관리자가 처음 솔루션에 접속한 뒤 알아야 할 기본 사항을 다룹니다. 세부 기능 운영은 이후 챕터에서 다룹니다.

## 접속

관리자 콘솔은 직접 URL로 진입하지 않습니다. 솔루션 좌상단의 4개 모드 전환 버튼 중 **관리 설정(Admin Settings)** 버튼을 클릭하면 관리자 영역으로 이동합니다.

1. 솔루션에 관리자 권한 계정으로 로그인
2. 화면 좌상단의 모드 전환 버튼(Agent 작업실 / AI 모델관리 / AI 거버넌스 / **관리 설정**) 중 **관리 설정** 클릭
3. 관리자 콘솔의 기본 진입점인 **환경 설정 → 전체 설정** 화면으로 이동

![관리 설정 진입 — 좌상단 4개 모드 전환 버튼에서 "관리 설정"을 클릭하면 환경 설정 → 전체 설정 화면으로 이동](images/admin-entry.png)

!!! note "관리자 권한 필요"
    관리자 콘솔의 모든 기능은 **관리자(Administrator)** 또는 **최고 관리자(Superuser)** 권한이 있는 계정에서만 사용 가능합니다. 일반 사용자에게는 메뉴 자체가 노출되지 않습니다.

## 권한 등급

본 솔루션은 3단계 권한 모델을 사용합니다.

| 등급 | 영문 | 가능한 작업 |
|---|---|---|
| 일반 사용자 | Standard User | Agent 작업실 사용. 자기 항목 편집·공유 |
| 관리자 | Administrator | 사용자/역할 관리, 시스템 설정, 운영 모니터링 |
| 최고 관리자 | Superuser | 모든 영역 + 권한 위임, 시스템 초기화 |

신규 관리자는 기본적으로 **관리자(Administrator)** 등급으로 부여됩니다. **최고 관리자(Superuser)** 는 시스템 전체에 1명 이상 유지하되, 가능하면 별도 계정으로 분리해 일반 작업과 구분하는 것을 권장합니다.

## 관리자 콘솔 구성

관리 설정 사이드바는 다음 11개 섹션으로 구성됩니다 (사용자 권한에 따라 일부는 표시되지 않을 수 있음).

| 섹션 | 영문 | 주요 메뉴 | 본 매뉴얼 챕터 |
|---|---|---|---|
| 사용자 / 접근제어 | Users / Access Control | 사용자 관리, 역할/권한 관리 | [사용자 관리](21-user-management.md), [역할/권한 관리](22-role-permission.md) |
| Agent 운영 | Agent Operations | 업무기획, Agent 관리, 채팅 모니터링, 사용자 토큰, 노드 관리, 프롬프트 템플릿 | (별도 챕터 미수록) |
| 환경 설정 | Environment | 전체 설정, **LLM**, 인프라, **검색 / 임베딩**, 오디오, 가드레일, 사이드바 | [LLM 설정](23-llm-settings.md), [임베딩·벡터 검색 설정](24-embedding-settings.md) |
| 시스템 상태 | System Status | **시스템 모니터링**, 시스템 조회, 로그 조회 | [시스템 모니터](26-system-monitor.md) |
| 보안 & 감사 | Security & Audit | 보안 설정, **감사 로그**, 에러 로그 | [감사 로그](27-audit-log.md) |
| 데이터 관리 | Data Management | 데이터베이스, 스케줄 쿼리, 배치 작업, 데이터 감사 로그 | (별도 챕터 미수록) |
| MCP 관리 | MCP Management | **MCP 라이브러리**, MCP 운영/모니터링 | [MCP 라이브러리](28-mcp-market.md) |
| MLOps | MLOps | ML 모델 관리 | (별도 챕터 미수록) |
| AI 거버넌스 | AI Governance | AI 위험도 평가, 점검 이력 관리, **통제 정책 관리**, 서비스 변경 이력 | [PII 보호 정책](25-pii-policy.md), [AI 거버넌스 대시보드](29-governance-dashboard.md) |
| 서비스 운영 | Service Operations | 공지 게시판, 자주 묻는 질문, 1:1 관리자 문의 | (별도 챕터 미수록) |
| 지식 운영 | Knowledge Operations | 컬렉션 관리 | (별도 챕터 미수록) |

!!! info "메뉴 명칭 안내"
    화면상의 메뉴 이름은 솔루션 버전과 사용자 권한에 따라 일부 다를 수 있습니다. 본 매뉴얼은 {{product.name}} {{product.version}} 기준이며, 표에 굵게 표시된 메뉴가 본 매뉴얼이 다루는 핵심 항목입니다.


## 첫 사용 시 점검 체크리스트

신규로 솔루션을 배포받은 직후 운영팀이 확인해야 할 항목입니다. 각 항목의 자세한 절차는 해당 챕터를 참조하세요.

- [ ] **최고 관리자 계정 활성화** — 시스템 설치 시 생성된 초기 superuser 계정에 로그인이 가능한지 확인
- [ ] **LLM 프로바이더 연결** — 회사 정책에 맞는 LLM 프로바이더(OpenAI / Anthropic / 사내 vLLM 등) 선택, API 키 또는 엔드포인트 등록
- [ ] **임베딩 모델 + 벡터 DB 연결** — 지식 검색이 동작하려면 필수
- [ ] **PII 정책 검토** — 자동 마스킹 대상 항목 확인. 금융권은 추가 정규식 등록 권장
- [ ] **역할 정의** — 조직에 맞는 역할(예: "분석가", "운영자")을 정의 후 사용자에 할당
- [ ] **시스템 모니터 임계치** — 알림을 받을 CPU/메모리/디스크 임계치 설정
- [ ] **감사 로그 보존 정책** — 보존 기간이 관련 규정과 부합하는지 확인 (금융권 통상 5년 이상)

## 운영 원칙

관리자 콘솔에서 작업할 때 따라야 할 일반 원칙입니다.

1. **변경 전 영향 범위 확인** — 시스템 설정 변경은 모든 사용자에 즉시 영향. 특히 LLM·임베딩 설정 변경은 진행 중인 채팅·실행 결과에 영향을 줄 수 있음.
2. **감사 로그 인식** — 모든 관리자 작업은 [감사 로그]에 기록됩니다. 정기 검토 권장.
3. **권한 최소화** — 사용자에게는 업무에 필요한 최소한의 권한만 부여. 임시 권한 상승은 작업 후 회수.
4. **백업 우선** — 컬렉션·프롬프트 일괄 변경 전 현재 상태를 내보내기(Export)로 백업.

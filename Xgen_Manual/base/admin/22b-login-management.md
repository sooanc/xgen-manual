# 로그인 관리

본 챕터는 *현재 시스템에 로그인되어 있는 사용자 세션* 을 실시간으로 확인하고 필요 시 즉시 회수하는 절차를 다룹니다. 좌측 메뉴 **관리 설정 → 사용자 / 접근제어 → 로그인 관리** (view ID `admin-active-sessions`) 를 선택해 진입합니다.

![로그인 관리 — 활성 사용자·활성 세션·슈퍼관리자 카운트 카드와 사용자별 세션 테이블, 행별 강제 로그아웃 버튼](images/admin-active-sessions.png)

진입 직후 화면 상단에 *"현재 시스템에 로그인되어 있는 사용자 목록입니다. 필요 시 사용자별로 모든 세션을 즉시 회수할 수 있습니다."* 라는 안내가 표시됩니다.

## 화면 한눈에 { #overview }

| 영역 | 위치 | 동작 |
|---|---|---|
| 버튼 — **새로고침** | 상단 좌측 | 현재 세션 목록을 다시 불러옵니다. 강제 로그아웃 직후 상태 확인용. |
| 카드 — **활성 사용자** | 상단 좌측 | 현재 1개 이상 활성 세션을 가진 *사용자 수* 카운트. |
| 카드 — **활성 세션** | 상단 중앙 | 사용자 수와 별개로 *세션(브라우저·기기 단위) 총 개수*. 한 사용자가 여러 기기·탭에서 로그인했다면 활성 사용자 < 활성 세션 이 됩니다. |
| 카드 — **슈퍼관리자** | 상단 우측 | 현재 활성 세션 중 SuperUser 권한자의 *사용자 수*. 통제 책임이 있는 권한자가 몇 명 접속해 있는지 한눈에 확인. |
| 테이블 | 화면 본문 | 사용자별 1행 — 활성 세션 / 최근 로그인 / 마지막 IP / 강제 로그아웃 액션. |

## 세션 테이블 컬럼 { #table }

| 컬럼 | 설명 |
|---|---|
| **사용자** | 사용자명. SuperUser 인 경우 노란색 `SUPER` 배지가 우측에 함께 표시. |
| **이름** | 사용자 표시 이름 (등록되어 있지 않으면 `-`). |
| **부서** | 부서 정보. 미설정 시 `-`. |
| **활성 세션** | 해당 사용자의 현재 활성 세션 *개수* — 숫자가 *링크 스타일* 로 표시됩니다. 클릭하면 세션별 상세(브라우저·기기·로그인 시각 등)를 펼쳐 볼 수 있습니다. |
| **최근 로그인** | 가장 최근 로그인 시각. `2026. 5. 25. 13시 36분 49초` 형태로 초 단위까지 표시. |
| **마지막 IP** | 가장 최근 접속 IP. 미기록 시 `-`. |
| **동작** | 행 우측 **강제 로그아웃** 버튼 — 해당 사용자의 *모든 활성 세션* 을 즉시 만료시킵니다. |

## 강제 로그아웃 { #force-logout }

특정 사용자의 모든 세션을 즉시 만료시키려면 행 우측 **강제 로그아웃** 버튼을 클릭합니다.

**처리 시점**:

- 클릭 즉시 해당 사용자의 *모든* 활성 세션이 무효화됩니다 (탭·기기 무관).
- 해당 사용자의 다음 API 요청부터 인증 실패가 반환되며, 브라우저는 자동으로 로그인 화면으로 리다이렉트됩니다.
- 강제 로그아웃은 *세션만* 종료하며 *계정 자체* 는 활성 상태로 유지됩니다. 본인은 다시 로그인할 수 있습니다.

**활용 사례**:

- 사용자가 *공용 PC 에서 로그아웃을 잊은 채 자리를 비웠다* 는 신고가 접수된 경우.
- 비밀번호 재설정 직후 *기존 세션을 끊고 새 비밀번호로 재로그인* 시키고 싶을 때 ([사용자 관리 · 비밀번호 재설정](21-user-management.md#비밀번호-재설정) 참고).
- 의심스러운 IP 에서 로그인된 세션이 보일 때 — *마지막 IP* 컬럼으로 식별 후 즉시 회수.
- 인사 변동(퇴사·전배) 처리 직전 — 강제 로그아웃 후 [사용자 관리](21-user-management.md#사용자-비활성화) 에서 *비활성* 처리.

!!! warning "강제 로그아웃은 즉시 적용·되돌릴 수 없음"
    클릭 즉시 처리되며 별도 확인 단계가 없으므로 신중하게 사용하세요. 일시적 차단이 필요하면 *비활성* 처리가 더 안전합니다 — 비활성은 *로그인 자체* 를 차단해 다시 로그인하지도 못합니다.

## 운영 권장사항

- **장시간 미사용 세션 정기 점검** — *활성 세션* 수가 *활성 사용자* 수에 비해 비정상적으로 많으면 사용자가 로그아웃 없이 닫은 탭·기기가 누적된 신호입니다. 주기적으로 새로고침해 추적하세요.
- **SuperUser 세션 별도 추적** — *슈퍼관리자* 카드 수치는 *관리자급 권한자가 현재 몇 명 접속해 있는가* 를 보여 줍니다. 야간·휴일 시간대에 평소보다 많으면 즉시 *마지막 IP* 컬럼과 함께 확인합니다.
- **비밀번호 재설정 직후 동반 강제 로그아웃** — 비밀번호 재설정만으로는 *이미 발급된 세션* 이 자동 만료되지 않습니다. 보안 사고 대응 시에는 *재설정 + 강제 로그아웃* 을 함께 수행하세요.
- **퇴사·전배 처리 표준화** — 인사 변동 워크플로우에 *강제 로그아웃 → 사용자 비활성화 → (필요 시) 권한 회수* 순서를 표준 절차로 등록.

## 자주 발생하는 문제

| 증상 | 원인 / 점검 |
|---|---|
| "강제 로그아웃을 눌렀는데 사용자가 여전히 사용 중이라고 합니다" | *목록 새로고침* 으로 실제 세션이 끊겼는지 확인. 사용자가 *이미 새로 로그인* 했다면 새 세션이 다시 생성됨 — *비활성화* 가 필요한지 검토. |
| "활성 세션 수가 활성 사용자 수보다 훨씬 많습니다" | 사용자가 로그아웃 없이 탭·기기를 닫은 경우 누적됩니다. 시스템 측 세션 만료 시간([환경 설정](20-admin-overview.md#env-overview)) 도 함께 검토. |
| "SuperUser 카드 수치가 갑자기 늘었어요" | 새로 부여된 SuperUser 가 있는지 확인 — [사용자 관리](21-user-management.md) 에서 사용자 유형이 *Superuser* 인 계정의 *마지막 로그인* 컬럼 점검. |
| "강제 로그아웃 버튼이 비활성화돼 있어요" | 본인 행의 *강제 로그아웃* 은 의도적으로 비활성. 본인 세션은 우상단 *Logout* 으로 종료하세요. |

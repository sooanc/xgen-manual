# 감사 로그

본 챕터는 사용자 활동·시스템 이벤트의 보존 기록인 감사 로그를 다룹니다. 금융권 등 규제 산업에서 정기 검토가 필수입니다.

## 기록되는 이벤트

| 이벤트 분류 | 예시 |
|---|---|
| 인증 | 로그인 성공/실패, 비밀번호 변경, 세션 만료 |
| 사용자 관리 | 사용자 추가·삭제·권한 변경, 역할 할당/회수 |
| 시스템 설정 | LLM·임베딩·정책 변경 |
| 콘텐츠 작업 | 에이전트플로우 생성·삭제·배포, 컬렉션 업로드 |
| 운영 | 시스템 시작/종료, 백업 실행 |

각 이벤트는 다음 정보를 포함합니다.

- 시각 (밀리초 단위 ISO 8601)
- 사용자 (또는 `system`)
- 이벤트 유형 (예: `user.role_changed`)
- 대상 자원 (예: `user:hong-gildong`)
- 변경 전/후 값 (해당 시)
- IP 주소, User-Agent

## 감사 로그 조회

좌측 메뉴 **관리 설정 → 보안 & 감사 → 감사 로그**를 선택합니다.

![데이터 감사 로그 — 필터 영역(기간·사용자·이벤트 유형·결과·IP)과 이벤트 목록](images/admin-data-audit-logs.png)

### 필터

| 필터 | 옵션 |
|---|---|
| 기간 | 최근 1시간 / 24시간 / 7일 / 30일 / 사용자 지정 |
| 사용자 | 특정 사용자명으로 필터 |
| 이벤트 유형 | 다중 선택 가능 |
| 결과 | 성공 / 실패 |
| IP | 특정 IP 대역 |

상단 검색창은 텍스트 검색(부분 일치)을 지원합니다 — 예: `hong` 입력 시 사용자명·대상에 hong이 포함된 모든 이벤트가 표시됩니다.

## 보존 기간

기본 보존 기간은 환경에 따라 다음과 같이 설정합니다.

| 환경 | 권장 보존 기간 | 근거 |
|---|---|---|
| 일반 기업 | 1~3년 | 내부 정책 |
| 금융권 | 5년 이상 | 전자금융감독규정 §17 |
| 공공기관 | 3~5년 | 정보통신망법, 개인정보보호법 |

보존 기간은 **관리 설정 → 보안 & 감사 → 보안 설정**의 감사 로그 보존 항목에서 설정합니다. 변경 자체도 감사 로그에 기록됩니다.

!!! warning "보존 기간 단축 시 주의"
    보존 기간을 줄이면 그 기간 이전 로그가 자동 삭제될 수 있습니다. 변경 전 반드시 **내보내기**로 백업하세요.

## 내보내기

규제 보고·외부 감사 대응을 위해 로그를 파일로 내보낼 수 있습니다.

1. 감사 로그 화면에서 필터 적용
2. 우상단 **내보내기** 버튼 → 형식 선택
    - CSV: 스프레드시트 분석용
    - JSON: 기계 처리용
3. 다운로드된 파일은 **암호화 보관**을 권장 (PII 포함 가능)

!!! info "현재 stg 빌드에 *내보내기* 버튼은 노출되지 않음"
    매뉴얼 이전 버전이 안내한 *우상단 "내보내기" 버튼* 은 현재 stg 라이브 빌드의 감사 로그 화면(`admin?view=admin-audit-logs`)에 노출되지 않습니다. 규제 보고가 필요한 경우 운영팀에 별도 데이터 추출(API/SQL/스크립트) 절차를 문의하세요. UI 노출이 추가되면 형식·범위 선택 모달 캡처와 함께 본 절을 보강합니다.

## 정기 검토 체크리스트

월 1회 권장 검토 항목입니다.

- [ ] 비정상 로그인 패턴 (반복 실패, 비업무 시간대 접속)
- [ ] 권한 등급 변경 이력 — 의도된 변경인지 확인
- [ ] LLM·임베딩 설정 변경 이력 — 변경 사유와 일치하는지
- [ ] 에이전트플로우 대량 삭제 이력
- [ ] 데이터 내보내기 빈도

## 운영 권장사항

- **자동 알림 연계** — 보안팀 SIEM과 연동하여 위험 이벤트(예: 권한 상승, 대량 삭제)는 실시간 알림.
- **정기 백업** — 분기별로 전체 감사 로그를 외부 스토리지로 백업. 시스템 장애 시 복구 가능.
- **접근 권한 제한** — 감사 로그 조회·내보내기 권한은 **SuperUser** 또는 **컴플라이언스 담당** 역할에만 부여 권장.

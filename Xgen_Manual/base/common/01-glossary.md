# 용어 정의

본 매뉴얼에서 사용되는 핵심 용어를 정의합니다. 각 기능별 세부 용어는 해당 챕터에서 다시 다룹니다.

## 솔루션 기본

| 용어 | 영문 | 정의 |
|---|---|---|
| Xgen | Xgen | 자연어 기반 업무 자동화와 다양한 시스템 연계를 통해 기업의 AI 실행 체계를 효율적으로 확장할 수 있는 Agentic AI Platform의 제품명 |
| Agent 작업실 | Agent Workspace | 에이전트와 에이전트플로우를 만들고 운영하는 사용자 작업 공간 |
| 관리 설정 | Admin Center | **SuperUser** 권한을 가진 사용자만 접근 가능한 관리자 영역. 혹은 '관리 설정' 페이지 내 특정 메뉴 권한을 가진 사용자만 접근 가능 |
| 대시보드 | Dashboard | 현황·통계를 한눈에 보는 시작 화면 |

## 핵심 객체

| 용어 | 영문 | 정의 |
|---|---|---|
| 에이전트 | Agent | 특정 작업을 자동화하기 위해 정의된 AI 작업 단위 |
| 에이전트플로우 | Agentflow | 여러 노드를 연결하여 정의한 AI 작업 흐름. 솔루션의 핵심 산출물 단위 |
| 노드 | Node | 에이전트플로우 내 하나의 작업 단위 (LLM 호출, 도구 실행, 분기 등) |
| 캔버스 | Canvas | 에이전트플로우를 드래그&드랍 및 시각적으로 편집하는 영역 |
| 도구 | Tool | 에이전트가 호출할 수 있는 외부 함수·API |
| MCP | Model Context Protocol | AI 모델과 외부 도구를 연결하는 표준 프로토콜 |
| 프롬프트 | Prompt | AI 모델에 전달하는 명령·지시문 (System/User Prompt 구분) |
| 인증 프로필 | Authentication Profile | 외부 API 연동에 필요한 인증 정보(키·토큰)를 묶어 관리하는 단위 |

!!! note "에이전트 vs 에이전트플로우"
    - **에이전트(Agent)**: 단일 작업을 수행하는 AI 작업 단위
    - **에이전트플로우(Agentflow)**: 여러 에이전트·노드를 엮은 워크플로우 — 운영 배포의 단위

## 지식 관리

| 용어 | 영문 | 정의 |
|---|---|---|
| 컬렉션 | Collection | 문서를 그룹화한 지식 저장 단위 |
| 문서 | Document | 컬렉션에 업로드된 텍스트·PDF·이미지 등 원본 자료 |
| 청크 | Chunk | 임베딩 처리를 위해 문서를 분할한 작은 텍스트 조각 |
| 임베딩 | Embedding | 청크를 검색 가능한 벡터로 변환한 결과 |
| 벡터 데이터베이스 | Vector Database | 임베딩 벡터를 저장·검색하는 DB. Qdrant 등의 엔진 사용 |
| 임베딩 모델 | Embedding Model | 텍스트를 벡터로 변환하는 모델 |
| 리랭커 | Reranker | 1차 벡터 검색 결과의 순위를 재조정하는 모델 |
| 온톨로지 | Ontology | 문서 내 개념과 관계를 구조화한 메타데이터 |

## 실행과 채팅

| 용어 | 영문 | 정의 |
|---|---|---|
| 채팅 | Chat | 사용자가 에이전트와 주고받는 대화 세션 |
| 메시지 | Message | 채팅에서의 한 회분 발언 (사용자 또는 에이전트) |
| 실행 | Execution | 에이전트플로우가 한 번 실행되어 결과를 산출한 사례 |
| 도구 호출 | Tool Call | 에이전트가 외부 도구·함수를 호출한 사건 |
| 인용 | Citations | AI 응답이 참고한 문서·자료 출처 |
| 로그 | Logs | 실행 과정의 단계별 상세 기록 |

## 배포와 운영

| 용어 | 영문 | 정의 |
|---|---|---|
| 배포 | Deploy | 에이전트플로우를 사용자 또는 외부 시스템이 호출 가능하도록 게시 |
| 임베드 | Embed | 외부 웹페이지에 에이전트플로우 인터페이스를 삽입하는 코드 |
| 버전 | Version | 에이전트플로우·프롬프트·문서의 변경 이력 단위 |
| 스케줄 | Schedule | 에이전트플로우 자동 실행 일정 (매일·매주·Cron 등) |
| 공유 | Share | 다른 사용자에게 항목 접근권을 부여 |
| 배포 상태 | Deployment Status | 에이전트플로우의 배포 단계 — 초안 / 활성 / 비활성 / 보관됨 |

## 사용자와 권한

| 용어 | 영문 | 정의 |
|---|---|---|
| 일반 사용자 | Standard User | 기본 사용자 권한 등급 — Agent 작업실 사용자 두 유형(**일반 사용자** / **Agent 개발자**) 을 포괄. 일반 사용자는 Agent 채팅·기술지원·대시보드를 사용하고, Agent 개발자는 여기에 Agent 제작·도구 연동·지식관리 메뉴 권한이 추가로 부여됨. 좌상단 **관리 설정** 모드는 특정 메뉴별 권한 부여 시 접근 가능 |
| 슈퍼유저 | SuperUser | 관리 권한 — 좌상단 **관리 설정** 모드 접근, 사용자·역할·시스템 설정·AI 거버넌스 등 모든 관리 기능 수행 |
| 역할 | Role | 권한 묶음 단위 (예: "시스템 관리자", "거버넌스 담당자", "운영자", "분석가") |
| 권한 | Permission | 특정 기능·자원에 대한 접근 허용 |

!!! note "권한 등급(2단계) vs 역할(여러 개) vs 권한(ABAC 키)"
    솔루션의 권한 체계는 **3 레이어** 로 분리됩니다.

    - **권한 등급(Tier)**: 2단계 — Standard User / SuperUser.
    - **역할(Role)**: 시스템 관리자, 거버넌스 담당자, 분석가 등 조직이 자유롭게 정의하는 라벨. 동일 SuperUser 등급 위에서도 역할에 따라 사이드바 메뉴 범위가 갈립니다.
    - **권한(Permission)**: `admin.user:read` 같은 **ABAC 키**. 역할에 묶이거나 사용자에게 직접 부여돼 화면·버튼 단위로 세밀하게 게이팅합니다.
        - *ABAC* = **A**ttribute-**B**ased **A**ccess **C**ontrol (속성 기반 접근 제어). 사용자의 속성(역할·권한 등) 을 기준으로 접근을 판정하는 모델이며, 여기서는 `resource:action` 형식의 문자열이 그 속성이 됩니다.

    전체 구조와 게이트 동작 원리는 [역할/권한 관리 · 권한 모델](../admin/22-role-permission.md#permission-model) 에서 자세히 설명합니다.

## 거버넌스와 보안

| 용어 | 영문 | 정의 |
|---|---|---|
| PII | Personally Identifiable Information | 개인을 식별할 수 있는 정보 (주민번호·전화·이메일 등) |
| 마스킹 | Masking | PII를 별표(`*`) 등으로 가려 표시하는 처리 |
| 위험 등급 | Risk Level | AI 응답·요청의 위험성 분류 |
| 감사 로그 | Audit Log | 사용자 활동·시스템 이벤트의 보존 기록 |

## 표기 규칙

본 매뉴얼은 다음 표기 규칙을 따릅니다.

- **한글 우선, 영문 병기**: 처음 등장 시 `에이전트플로우(Agentflow)` 형태로 영문을 괄호 부기. 같은 챕터 내 두 번째부터는 한글만.
- **혼용 표기 통일**: 코드상 `Canvas`/`캔버스`처럼 혼용되는 단어는 매뉴얼에서 한글로 통일.
- **코드·식별자**: 등폭 글꼴 — 예: `customer.id`, `xgen.example.com`.
- **언어 전환**: 본 매뉴얼은 한국어가 기본입니다. 화면 우측 상단의 언어 토글(🌐)로 영문판으로 전환 가능합니다.

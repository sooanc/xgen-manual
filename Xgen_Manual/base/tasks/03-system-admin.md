# 시스템 관리자

> 솔루션 인프라·사용자·인증·LLM 연결을 **운영**하는 사용자. 관리 설정 모드의 대부분을 사용합니다.

## 로그인 직후 내 역할에 맞는 메인 화면을 보고 싶다
- 시작 위치: 로그인 직후 자동 진입 (또는 좌상단 **XGEN** 로고)
- 절차: [대시보드 · 시스템 관리자 뷰](../admin/30-dashboard.md#시스템-관리자-뷰)
- 시스템 관리자에게는 "System Operations & Deployment Dashboard" 와 함께 **관리 설정** 단축 버튼 활성화, **시스템 상태·운영 지표** 위젯이 노출됩니다.

## 솔루션을 처음 배포받았다, 무엇부터 확인하나
- [관리자 콘솔 개요 - 첫 사용 시 점검 체크리스트](../admin/20-admin-overview.md#첫-사용-시-점검-체크리스트)
- 핵심: superuser 활성화 → LLM 프로바이더 → 임베딩·벡터 DB → 역할 정의

## 새 사용자를 추가하거나 비활성화하고 싶다
- 시작 위치: 관리 설정 → 사용자 / 접근제어 → 사용자 관리
- 절차: [사용자 관리](../admin/21-user-management.md)

## 사용자 권한을 조정하고 싶다 (역할 추가·할당)
- 시작 위치: 관리 설정 → 사용자 / 접근제어 → 역할/권한 관리
- 절차: [역할/권한 관리](../admin/22-role-permission.md)

## LLM 프로바이더(OpenAI / Anthropic / 사내 vLLM)를 등록하려면
- 시작 위치: 관리 설정 → 환경 설정 → LLM
- 절차: [LLM 설정](../admin/23-llm-settings.md)

## 임베딩 모델 + 벡터 DB를 연결하려면 (지식 검색 활성화)
- 시작 위치: 관리 설정 → 환경 설정 → 검색 / 임베딩
- 절차: [임베딩·벡터 검색 설정](../admin/24-embedding-settings.md)

## CPU·메모리·디스크 상태를 모니터링하려면
- 시작 위치: 관리 설정 → 시스템 상태 → 시스템 모니터링
- 절차: [시스템 모니터](../admin/26-system-monitor.md)
- 알림 임계치 설정 권장

## Agent 개발자가 만든 에이전트의 배포 요청을 승인·거부하려면 (1차 배포 승인)
- 시작 위치: 관리 설정 → **Agent 운영 → Agent 관리**
- 절차: [Agent 관리 — 배포 승인 (시스템 관리자 1차 승인)](../admin/32-agent-operations.md#agent-mgmt-deploy-approval)
- 핵심: **배포 대기** 배지가 붙은 카드(`inquire_deploy: true`) 의 **⋯** 메뉴에서 **승인** 또는 **거부** 선택. 승인 시 거버넌스 큐로 자동 진행 — 사용자 노출은 거버넌스 승인까지 통과해야 합니다.
- 위젯: 대시보드 *Agent 배포/승인 상태* 의 "배포 승인 대기" 카운트로 일일 큐 점검.

## 배포 승인된 에이전트가 운영 중 문제가 생겼다 (긴급 차단)
- 시작 위치: 관리 설정 → Agent 운영 → Agent 관리 → 대상 카드의 **설정** 아이콘
- 절차: 에이전트플로우 설정 모달의 **승인 상태(Approval Status)** 토글을 *비활성* 으로 전환
- 효과: 즉시 실행 차단(킬 스위치). 같은 모달의 **배포 토글**(공개/비공개) 과 함께 단계적 롤백 가능. 자세히는 [Agent 운영](../admin/32-agent-operations.md) 참고.

## MCP 서버를 등록·운영하려면
- 시작 위치: 관리 설정 → MCP 관리 → MCP 라이브러리
- 절차: [MCP 라이브러리](../admin/28-mcp-market.md)

## 사용자가 관리자 문의 게시판·1:1 문의를 보냈다, 답변하려면
- 시작 위치: 관리 설정 → 서비스 운영
- 절차: [기술지원 응대](../admin/31-tech-support-handling.md)

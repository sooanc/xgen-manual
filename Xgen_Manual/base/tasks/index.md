# 업무 가이드

본 페이지는 사용자가 **하고 싶은 일** 기준으로 매뉴얼을 탐색하도록 돕습니다. 메뉴를 외울 필요 없이 자기 역할과 업무 목적으로 들어가세요.

> 메뉴 구조 기준의 상세 레퍼런스가 필요하면 상단의 [공통](../common/) / [사용자](../user/) / [관리자](../admin/) 탭으로 이동하세요.

## 어떤 역할이신가요?

| 역할 | 주요 관심사 | 바로가기 |
|---|---|---|
| 일반 사용자 | Agent 실행·채팅·지식 활용 | [↓ 일반 사용자](#일반-사용자) |
| Agent 개발자 | Agent 설계·배포·승인 | [↓ Agent 개발자](#agent-개발자) |
| 시스템 관리자 | 사용자·권한·LLM·시스템 운영 | [↓ 시스템 관리자](#시스템-관리자) |
| 거버넌스 담당자 | 승인·통제 정책·감사 | [↓ 거버넌스 담당자](#거버넌스-담당자) |

---

## 일반 사용자

> 회사가 미리 배포해둔 Agent를 **채팅으로 활용**하는 사용자. 사용 가능한 메뉴는 **Agent 채팅 / 기술지원 / 대시보드(개인용 영역)** 셋입니다. 그 외 메뉴(Agent 제작·도구 연동·지식관리)는 Agent 개발자 권한이 필요합니다.

### Agent와 채팅하고 싶다
- 시작 위치: 좌측 사이드바 **Agent 채팅 → 새 채팅**
- 절차: [채팅 사용](../user/14-chat.md)

### 내 사용 통계가 궁금하다 (개인용 대시보드)
- 시작 위치: 상단 헤더 **대시보드**
- 절차: [대시보드](../user/18-dashboard.md)
- 표시 범위: 본인 채팅 횟수·자주 쓴 Agent·최근 활동 등 **개인 사용 한정**. 거버넌스·시스템 운영 위젯은 관리자 권한이 있어야 노출됨.

### 공지·FAQ를 찾거나 문의하고 싶다
- 시작 위치: 좌측 사이드바 **기술지원**
- 절차: [기술지원](../user/19-tech-support.md)

### 솔루션 화면이 처음이다, 어디부터 봐야 하나
- [시작하기](../user/10-getting-started.md) → [로그인](../user/11-login.md)

> Agent를 *만들거나* 지식·프롬프트·인증 프로필을 관리하려면 [Agent 개발자](#agent-개발자) 절을 참고하세요.

## Agent 개발자

> 자기 업무를 자동화할 Agent를 **직접 설계·배포**하는 사용자. 일반 사용자 메뉴에 더해 **Agent 제작·도구 연동·지식관리** 영역 사용 권한이 부여됩니다.

### 첫 Agent를 만들고 배포하는 전체 흐름

1. **작업실 진입** — Agent 작업실 → Agent 설계 → [Agent 작업실 진입](../user/12-agentflow-create.md#agent-작업실-진입)
2. **노드 구성** — Start Node 부터 캔버스에 노드 추가·연결 → [에이전트플로우 만들기](../user/12-agentflow-create.md)
3. **외부 도구 연결** — API Tool / MCP 노드 등록 → [API 도구](../user/12-agentflow-create.md#노드-추가)
4. **품질 평가** — 테스트 데이터로 응답 품질 확인 → [에이전트플로우 운영](../user/13-agentflow-operations.md#에이전트-품질-분석)
5. **배포** — 운영 환경에 배포 → [에이전트플로우 운영](../user/13-agentflow-operations.md)
6. **승인 대기** (위험도 임계치 초과 시) — 거버넌스 담당자 승인 후 노출 → [AI 거버넌스 - 위험도 평가 및 심사](../admin/29-governance-dashboard.md#ai-위험도-평가-및-심사)

### Agent에게 사내 문서를 참고하게 하고 싶다 (RAG)
- 시작 위치: 좌측 사이드바 **지식관리 → 컬렉션**
- 절차: [지식관리](../user/15-knowledge.md)

### 자주 쓰는 프롬프트를 저장·공유하고 싶다
- 시작 위치: 좌측 사이드바 **Agent 제작 → Agent 프롬프트**
- 절차: [프롬프트 관리](../user/16-prompt.md)

### 외부 시스템(API) 자격증명을 안전하게 관리하고 싶다
- 시작 위치: 좌측 사이드바 **도구 연동 → 인증 프로필**
- 절차: [인증 프로필](../user/17-auth-profile.md)

### 내 Agent가 거버넌스 승인 대기 중이다
- 시작 위치: 관리 설정 → AI 거버넌스 → 에이전트 승인
- 절차: [AI 거버넌스 - 위험도 평가 및 심사](../admin/29-governance-dashboard.md#ai-위험도-평가-및-심사)
- 참고: 본인이 승인 권한이 없다면 거버넌스 담당자에게 검토 요청

### 외부 API/MCP 서버를 도구로 등록하고 싶다
- 시작 위치: 좌측 사이드바 **도구 연동 → API Tool**
- 절차: [에이전트플로우 만들기 - 노드 추가](../user/12-agentflow-create.md#노드-추가)

### 운영 중인 Agent의 호출 통계·실패율을 보고 싶다
- 시작 위치: 헤더 **대시보드** 또는 사이드바 **Agent 운영 설정**
- 절차: [에이전트플로우 운영](../user/13-agentflow-operations.md)

## 시스템 관리자

> 솔루션 인프라·사용자·인증·LLM 연결을 **운영**하는 사용자. 관리 설정 모드의 대부분을 사용합니다.

### 솔루션을 처음 배포받았다, 무엇부터 확인하나
- [관리자 콘솔 개요 - 첫 사용 시 점검 체크리스트](../admin/20-admin-overview.md#첫-사용-시-점검-체크리스트)
- 핵심: superuser 활성화 → LLM 프로바이더 → 임베딩·벡터 DB → 역할 정의

### 새 사용자를 추가하거나 비활성화하고 싶다
- 시작 위치: 관리 설정 → 사용자 / 접근제어 → 사용자 관리
- 절차: [사용자 관리](../admin/21-user-management.md)

### 사용자 권한을 조정하고 싶다 (역할 추가·할당)
- 시작 위치: 관리 설정 → 사용자 / 접근제어 → 역할/권한 관리
- 절차: [역할/권한 관리](../admin/22-role-permission.md)

### LLM 프로바이더(OpenAI / Anthropic / 사내 vLLM)를 등록하려면
- 시작 위치: 관리 설정 → 환경 설정 → LLM
- 절차: [LLM 설정](../admin/23-llm-settings.md)

### 임베딩 모델 + 벡터 DB를 연결하려면 (지식 검색 활성화)
- 시작 위치: 관리 설정 → 환경 설정 → 검색 / 임베딩
- 절차: [임베딩·벡터 검색 설정](../admin/24-embedding-settings.md)

### CPU·메모리·디스크 상태를 모니터링하려면
- 시작 위치: 관리 설정 → 시스템 상태 → 시스템 모니터링
- 절차: [시스템 모니터](../admin/26-system-monitor.md)
- 알림 임계치 설정 권장

### MCP 서버를 등록·운영하려면
- 시작 위치: 관리 설정 → MCP 관리 → MCP 라이브러리
- 절차: [MCP 라이브러리](../admin/28-mcp-market.md)

### 사용자가 기술지원 게시판·FAQ·1:1 문의를 보냈다, 답변하려면
- 시작 위치: 관리 설정 → 서비스 운영
- 절차: [기술지원 응대](../admin/31-tech-support-handling.md)

## 거버넌스 담당자

> AI 사용에 대한 **위험·통제·감사**를 책임지는 사용자. 일반 시스템 관리자와 권한이 분리되는 게 일반적입니다.

### 승인 대기 중인 Agent들을 검토·승인하려면
- 시작 위치: 관리 설정 → AI 거버넌스 → AI 위험도 평가 및 심사 → 에이전트 승인
- 절차: [AI 거버넌스 - 위험도 평가 및 심사](../admin/29-governance-dashboard.md#ai-위험도-평가-및-심사)
- 검토 항목: 노드 구성, 권한 범위, PII 영향도

### 위험도 평가 카테고리·가중치를 조정하려면
- 시작 위치: 관리 설정 → AI 거버넌스 → AI 통제 정책 관리
- 절차: [AI 거버넌스 - 통제 정책 관리](../admin/29-governance-dashboard.md#ai-통제-정책-관리)
- 권장: 분기 1회 재조정

### PII 마스킹 규칙을 추가하거나 조정하려면
- 시작 위치: 관리 설정 → AI 거버넌스 → AI 통제 정책 관리 → PII 정책
- 절차: [PII 보호 정책](../admin/25-pii-policy.md)

### 사용자 행위 (로그인·시스템 설정 변경 등)를 추적하려면
- 시작 위치: 관리 설정 → 보안 & 감사 → 감사 로그
- 절차: [감사 로그](../admin/27-audit-log.md)
- 일반 시스템 행위 감사

### 거버넌스 정책·승인 변경 이력을 추적하려면
- 시작 위치: 관리 설정 → AI 거버넌스 → AI 감사·추적 관리
- 절차: [AI 거버넌스 - 감사·추적 관리](../admin/29-governance-dashboard.md#ai-감사추적-관리)
- 거버넌스 한정 감사 (일반 시스템 로그와 분리)

### 정기 점검 일정을 등록·관리하려면
- 시작 위치: 관리 설정 → AI 거버넌스 → AI 점검 이력 및 계획
- 절차: [AI 거버넌스 - 점검 이력 및 계획](../admin/29-governance-dashboard.md#ai-점검-이력-및-계획)

### 거버넌스 종합 모니터링이 궁금하다
- 시작 위치: 관리 설정 → AI 거버넌스 (기본 진입 화면)
- 절차: [AI 거버넌스](../admin/29-governance-dashboard.md)

# 업무 기획

본 챕터는 사용자가 "이 업무를 자동화할 Agent 가 필요하다" 는 **아이디어 단계의 기획서**를 등록·관리하는 화면을 다룹니다. 좌측 사이드바 **분석 / 기획 → 업무기획** 메뉴가 본 챕터 범위입니다.

> 본 화면은 **Agent 를 만들기 전** 단계입니다. 실제 캔버스에서 에이전트를 조립하는 절차는 [에이전트 만들기](12-agentflow-create.md) 챕터를, 만든 에이전트를 운영하는 절차는 [에이전트 운영](13-agentflow-operations.md) 챕터를 참고하세요.

## 화면 진입

좌측 사이드바 **분석 / 기획 → 업무기획** 을 선택합니다.

![업무 기획 — '작성한 Agent 개발 기획서 목록' 헤더, 검색·영향 범위 필터, '새 기획서 제작하기' 버튼. 표 컬럼은 분석 ID·기획서 제목·업무 분야·영향 범위·연동 Agent·최종 수정](images/main-planning.png)

## 화면 구성

| 영역 | 내용 |
|---|---|
| 상단 | 화면 제목 "업무기획" 과 안내 ("작성한 Agent 개발 기획서 목록입니다.") |
| 검색·필터 | 제목·연동 Agent 검색, **영향 범위** 필터(전사 / 소속 부서 / 본인) |
| 우상단 | **새 기획서 제작하기** — 신규 기획서 작성 |
| 본문 표 | 분석 ID · 기획서 제목 · 업무 분야 · 영향 범위 · 연동 Agent · 최종 수정 일자 |

## 기획서 등록

새 기획서는 화면 우상단 **새 기획서 제작하기** 버튼으로 시작합니다. 입력 항목은 다음과 같습니다.

| 항목 | 설명 |
|---|---|
| 기획서 제목 | 한 줄로 무엇을 자동화할 Agent 인지 표현 (예: "주간 영업 KPI 자동 집계 Agent") |
| 업무 분야 | 영업 / 영업기획 / 전략기획 / 고객서비스 / 컴플라이언스 / 리스크 / 여신 등 조직 분류 |
| 영향 범위 | **전사** / **소속 부서** / **본인** 중 선택 — 협업·승인 범위와 직접 연결 |
| 연동 Agent | 본 기획이 활용·확장하는 기존 에이전트 (선택사항) |
| 본문 | 배경·요구사항·예상 입출력·KPI 등 상세 (마크다운 지원) |

저장 후에는 표에 분석 ID 와 함께 행이 추가되어 동료가 검색·열람할 수 있습니다.

## 권장 흐름

1. **기획서 먼저 작성** — 만들기 전에 아이디어를 글로 정리하면 캔버스에서의 노드 구성도 빨라집니다.
2. **연관 Agent 표기** — 비슷한 일을 하는 기존 에이전트가 있다면 **연동 Agent** 에 묶어 두면 중복 개발을 피하거나 참조할 수 있습니다.
3. **영향 범위 신중 선택** — "전사" 로 등록하면 거버넌스 담당자의 검토 우선순위에 올라갑니다. 본인 사용 목적이면 "본인" 으로 두세요.

## 자주 발생하는 문제

- **"목록 조회 실패: Bad Gateway"** — 백엔드 일시 오류. 우상단 새로고침으로 재시도. 반복되면 [기술지원](19-tech-support.md) 으로 문의.
- **새 기획서 제작 버튼이 비활성** — 권한 부족. Agent 개발자 권한이 있어야 등록 가능합니다.
- **분석 ID 가 비어 있음** — 저장 직후 일시적으로 표시되지 않을 수 있음. 새로고침 후 정상 표시 확인.

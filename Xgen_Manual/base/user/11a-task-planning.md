# Agent 기획

본 챕터는 사용자가 "이 업무를 자동화할 Agent 가 필요하다" 는 **아이디어 단계의 기획서**를 등록·관리하는 화면을 다룹니다. 좌측 사이드바 **Agent 제작 → Agent 기획** 메뉴(Agent 제작 섹션 최상단) 가 본 챕터 범위입니다.

> 본 화면은 **Agent 를 만들기 전** 단계입니다. 실제 캔버스에서 에이전트를 조립하는 절차는 [에이전트 만들기](12-agentflow-create.md) 챕터를, 만든 에이전트를 운영하는 절차는 [에이전트 운영](13-agentflow-operations.md) 챕터를 참고하세요.

## 화면 진입 및 신규 기획서 작성

좌측 사이드바 **Agent 제작 → Agent 기획** 을 선택하면 작성한 Agent 개발 기획서 목록이 표시됩니다. 목록 우상단 **+ 새 기획서 제작하기** 버튼을 누르면 *Agent 개발 기획서* 작성 모달이 열립니다.

![Agent 기획 — 목록에서 '새 기획서 제작하기' 클릭 시 5단계 작성 모달이 열리는 흐름](images/plan-new.gif)

## 목록 화면 구성

| 영역 | 내용 |
|---|---|
| 상단 | 화면 제목 "업무기획" 과 안내 ("작성한 Agent 개발 기획서 목록입니다.") |
| 검색·필터 | 제목·연동 Agent 검색, **영향 범위** 필터(전체 / 전사 / 소속 부서 / 본인) |
| 우상단 | **+ 새 기획서 제작하기** — 신규 기획서 작성 모달 열기 |
| 본문 표 | 문서 ID · 기획서 제목 · 업무 분야 · 영향 범위 · 연동 Agent · 최종 수정 일자 |

## 기획서 작성 모달 — 5단계 구조

새 기획서는 단계별 탭으로 구성된 모달에서 작성합니다. 상단에 5단계 진행 표시가 있으며, 우상단 **취소 / 임시 저장 / 제출** 버튼으로 작업을 마무리합니다.

| 단계 | 입력 항목 |
|---|---|
| **01 기본 정보** | 기획서 제목, 업무 분야, 작성일, 연관 워크플로우, 연관 업무 기획서, 개발 유형(자체 개발 / 의뢰 개발) |
| **02 문제 정의** | 자동화 대상 업무의 현재 상태와 해결하고자 하는 문제 |
| **03 프로세스 분석** | 입출력 정의, 단계별 처리 흐름, 외부 시스템 연동 |
| **04 기대 효과** | 정량/정성 효과, KPI, 도입 후 변화 |
| **05 첨부파일** | 관련 자료(스펙, 데이터 샘플 등) 업로드 |

**임시 저장** 은 작성 중간에 닫아도 다음에 이어쓸 수 있게 보존합니다. **제출** 은 모든 단계를 확정하여 목록에 정식 기획서로 등록합니다.

### 기본 정보 절 — 주요 필드 보충

| 항목 | 설명 |
|---|---|
| 기획서 제목 | 한 줄로 무엇을 자동화할 Agent 인지 표현 (예: "주간 영업 KPI 자동 집계 Agent") |
| 업무 분야 | 영업 / 영업기획 / 전략기획 / 고객서비스 / 컴플라이언스 / 리스크 / 여신 등 조직 분류 |
| 영향 범위 | **전사** / **소속 부서** / **본인** — 협업·승인 범위와 직접 연결 |
| 연관 워크플로우 | 본 기획이 활용·확장하는 기존 에이전트플로우 (선택사항) |
| 연관 업무 기획서 | 관련 있는 다른 Agent 기획서 — 영향 분석과 협업에 도움 |
| 개발 유형 | **자체 개발** (본인이 직접 캔버스로 제작) / **의뢰 개발** (개발팀에 제작 의뢰) |

## 권장 흐름

1. **기획서 먼저 작성** — 만들기 전에 아이디어를 글로 정리하면 캔버스에서의 노드 구성도 빨라집니다.
2. **연관 워크플로우·기획서 표기** — 비슷한 일을 하는 기존 항목이 있다면 함께 묶어 두면 중복 개발을 피하거나 참조할 수 있습니다.
3. **영향 범위 신중 선택** — "전사" 로 등록하면 거버넌스 담당자의 검토 우선순위에 올라갑니다. 본인 사용 목적이면 "본인" 으로 두세요.
4. **단계 중간 임시 저장** — 5단계를 한 번에 완성하기 부담스러우면 *임시 저장* 으로 끊고 이어 작성합니다.

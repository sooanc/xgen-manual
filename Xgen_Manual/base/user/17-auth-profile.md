# 인증 프로필

본 챕터는 외부 API·시스템 연동에 필요한 인증 정보를 관리하는 **인증 프로필(Authentication Profile)** 사용법을 다룹니다.

## 인증 프로필이란

에이전트플로우가 외부 시스템(예: 사내 데이터베이스, 외부 API, Git 저장소)에 접근할 때 필요한 인증 정보(API 키, 액세스 토큰 등)를 한곳에 묶어 관리하는 단위입니다.

| 항목 | 설명 |
|---|---|
| 개별 인증값 | API 키, 토큰, 사용자명·비밀번호 |
| 프로필 | 위 인증값들을 한 묶음으로 관리 |

직접 노드 설정에 키를 입력하지 않고 프로필을 참조하므로, 키 변경 시 한 곳만 수정하면 모든 에이전트플로우에 반영됩니다.

## 인증 프로필 목록

좌측 사이드바에서 **도구 연동 → 인증 프로필** 메뉴를 선택합니다.

![인증 프로필 목록 — 등록된 프로필별 이름·종류·공개 범위·마지막 사용 정보](images/auth-profile.png)

각 프로필은 다음 정보를 가집니다.

| 컬럼 | 설명 |
|---|---|
| 이름 | 프로필 식별자 |
| 종류 | API 키 / OAuth / Basic Auth 등 |
| 공개 범위 | 비공개 (본인만) / 공유 |
| 마지막 사용 | 가장 최근 호출 시각 |

## 새 프로필 생성

1. 우상단 **+ 새 인증 프로필** 버튼
2. 다음 항목 입력
    - **이름**: 어디 인증인지 식별 가능하게 (예: "사내 ERP API")
    - **종류**: API 키 / OAuth / Basic Auth 등
    - **인증값**: 종류에 따라 다른 필드
3. **프로필 테스트** 클릭 → 실제 호출 성공 확인
4. **저장**

![인증 프로필 생성 — *새 프로필* 버튼 클릭 시 노출되는 종류 선택·인증값 입력 화면](images/auth-profile-create.png)

!!! info "버튼 라벨 안내"
    실제 stg 화면의 버튼 라벨은 **"새 프로필"** 입니다 (매뉴얼 이전 버전이 적은 *"+ 새 인증 프로필"* 과 다름).

## 프로필 종류

| 종류 | 입력 필드 | 사용처 |
|---|---|---|
| API 키 | API Key 한 개 | 대부분 외부 API |
| Basic Auth | 사용자명, 비밀번호 | 레거시 HTTP 기본 인증 |
| OAuth 2.0 | Client ID, Client Secret, 토큰 URL | OAuth 지원 서비스 |
| Bearer Token | 액세스 토큰 | 토큰 기반 API |

## 에이전트플로우에서 사용

에이전트플로우 캔버스에서 도구 노드(외부 API·DB 노드 등)를 추가할 때:

1. 노드 상세 패널의 **인증 프로필** 드롭다운
2. 등록된 프로필 중 선택
3. 저장

이후 노드는 선택된 프로필의 인증값을 자동으로 사용합니다.

## 프로필 공유

다른 사용자도 같은 인증 정보를 사용해야 한다면 공유:

1. 프로필 상세 → **공유** 버튼
2. 사용자·역할 선택 + 권한 (읽기 / 읽기·쓰기)
3. **저장**

!!! warning "공유 시 보안"
    프로필을 공유하면 해당 사용자도 그 인증 정보로 외부 호출이 가능해집니다. 공유 대상은 신중히 선택하고, 비밀번호급 인증값(루트 API 키 등)은 가능하면 공유하지 마세요.

## 키 변경

외부 시스템에서 키를 새로 발급받았다면:

1. 프로필 상세 → **편집**
2. 새 인증값 입력
3. **프로필 테스트** → **저장**

이 프로필을 사용하는 모든 에이전트플로우에 즉시 반영됩니다.

## 운영 권장사항

- **프로필 이름 규칙** — "환경-시스템-용도" 형식 권장 (예: `prod-erp-readonly`)
- **만료 추적** — OAuth 토큰은 만료가 있으므로 주기적으로 갱신하거나 자동 갱신 설정
- **키 회전** — 분기별로 키 회전하여 노출 위험 최소화. 변경 시점에 짧은 다운타임 가능

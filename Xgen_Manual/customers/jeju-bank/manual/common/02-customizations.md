# 커스터마이징 가이드

본 챕터는 고객사 환경에 맞춰 추가 개발·조정된 기능을 정리합니다. 표준 {{product.name}} 매뉴얼과 다른 부분만 다루며, 각 항목은 화면 경로와 함께 사용 시점을 안내합니다.

!!! info "스크린샷 보강"
    본 챕터에 명시된 스크린샷은 이후 회차에서 stg(`https://stg-xgen.x2bee.com`) 환경 기준으로 캡처해 보강합니다. 현재는 텍스트 기반 절차·라벨만 정리되어 있습니다.

## 1. Agent 기획 메뉴 위치 변경

업무 기획 화면이 사용자 사이드바 최상단으로 이동했습니다.

| 구분 | 표준 매뉴얼 | 고객사 환경 |
|---|---|---|
| 위치 | *분석 / 기획 → 업무기획* (별도 섹션) | **Agent 제작 → Agent 기획** (최상단) |
| 라벨 | "업무기획" | "Agent 기획" |
| 권한 | Agent 개발자 | Agent 개발자 (변동 없음) |

**의도**: 업무 기획서 작성과 캔버스에서의 Agent 설계를 하나의 사이드바 섹션으로 묶어, 기획 → 설계 → 운영 흐름을 한눈에 보이게 합니다.

**진입 경로**: `Agent 작업실 → Agent 제작 → Agent 기획`

## 2. 캔버스 채팅 빌더 — *채팅으로 워크플로우 만들기*

자연어 대화로 에이전트플로우 노드를 자동 구성하는 기능입니다.

| 항목 | 내용 |
|---|---|
| 진입 경로 | Agent 작업실 → Agent 제작 → Agent 설계 → 캔버스 상단 **채팅으로 만들기** 진입 |
| 대화 흐름 | 사용자가 자연어로 원하는 흐름을 설명 → AI 가 노드 구성을 제안 → 수락 시 캔버스에 즉시 반영 |
| 시각 표시 | 응답 스트리밍 중 커서 표시, 단계별 진행 상태 표시, 좌·우 여백 정돈 |
| 제어 | 진행 중 응답은 *중단* 가능, 생성된 노드는 일반 캔버스 편집과 동일하게 수정 가능 |

**활용 예시**:

- "사내 규정 문서를 검색해 답변하는 챗봇" → AI 가 *지식 컬렉션 조회 → LLM 응답 → 인용 부착* 노드 구성을 제안
- 기존 캔버스에 노드를 *추가로 끼워 넣는* 형태로도 사용 가능

## 3. 모델 서빙 멀티노드 — *서버 선택 UI*

모델 서빙 환경에서 다중 노드(서버)를 운용할 때 어느 서버에 모델을 로드·실행할지 선택할 수 있습니다.

| 항목 | 내용 |
|---|---|
| 진입 경로 | 관리 설정 → AI 모델관리 → 모델 서빙 → **모델 로드 모달** |
| 추가 입력 | 서버(노드) 선택 드롭다운, vLLM launch 인자 입력 필드, parser datalist |
| 안내 | 컨텍스트 OOM(메모리 초과) 경고 문구를 모달 내부에 노출 |
| 라우팅 | 선택한 서버 노드로 모델 호출 요청을 라우팅 |

**운영 시 주의**: 동일 모델을 여러 노드에 중복 로드할 경우 GPU 메모리 점유가 누적되므로, 사용 패턴에 맞춰 노드를 분담 배치하는 것을 권장합니다.

## 4. SSO 로그인

로그인 화면에 표준 이메일·비밀번호 입력 외에 **SSO 로그인** 버튼이 추가되어, 사내 ID 통합 인증 경로로 로그인할 수 있습니다.

| 항목 | 내용 |
|---|---|
| 진입 경로 | `https://{{product.domain}}/login` → 일반 로그인 폼 아래 **SSO 로그인** 버튼 |
| 동작 | 클릭 시 사내 SSO 엔드포인트(`/sso/signin`)로 리다이렉트 |
| 라벨 | 일반 로그인 폼과 SSO 버튼 사이에 *또는* 구분선 노출 |

**실패 시**: SSO 응답이 지연되거나 실패하면 일반 로그인 폼으로 돌아가 사번/이메일·비밀번호 로그인을 시도하세요. 사내 인증 시스템 측 문제가 의심되면 {{vars.support_email}} 로 즉시 알립니다.

## 5. 사이드바 빠른 이동 — *Teams · Canvas*

사이드바 헤더와 대시보드 환영 영역에 **Teams** / **Canvas** 빠른 이동 버튼이 추가되었습니다.

| 위치 | 형태 |
|---|---|
| 사이드바 헤더 (사용자·관리자 양쪽) | 작은 outline pill 버튼 |
| 대시보드 *환영 카드* 우측 | 큰 그라데이션 버튼 (chevron 아이콘) |

- **Teams** 버튼: 항상 표시되며, 권한이 있는 사용자만 활성. 클릭 시 `/teams` 로 이동.
- **Canvas** 버튼: `main.agentflow-canvas:read` 또는 `:manage` 권한이 있는 사용자에게만 노출. 클릭 시 `/canvas` 로 이동.

## 6. 대시보드 위젯 개편

### 6-1. 신규 위젯 — *Agent 현황* (시스템 관리자 뷰)

배포 승인 처리 큐를 즉시 파악할 수 있는 3-stat 위젯이 추가되었습니다.

| stat | 의미 |
|---|---|
| 배포 승인 대기 | 시스템 관리자 1차 승인 대기 건수 |
| 승인 반려 (완료 후) | 거버넌스 단계에서 반려된 건수 |
| 승인 반려 | 시스템 관리자 1차 단계에서 반려된 건수 |

권한 `admin.governance-review:read` 가 있어야 노출됩니다.

### 6-2. 위젯 라벨·구조 변경

| 위젯 | 이전 | 이후 |
|---|---|---|
| 점검 현황 | 점검 현황 (정상/주의/긴급) + 최근 5건 | **점검 기한 초과 현황** — 4-stat (초과 / 예정 / 완료 / 전체 대상 에이전트) |
| 재평가 현황 | 거부 카운트 + 거부된 에이전트 목록 | **위험 평가 재수행 대상** — 4-stat (신규 배포승인 / AI 미승인 건수 / 미승인 비율 / 승인 비율) |
| 위험 등급 분포 | 위험 등급 분포 | **등급별 활성 비율** |
| 내가 남긴 평가 | 내가 남긴 평가 | **내가 남긴 평가 (별점/코멘트)** |

### 6-3. 부서별 현황 테이블 시각화

부서별 현황 위젯이 *세로 리스트* 에서 **표 (4 컬럼)** 로 변경됐습니다. 활성 Agent 컬럼에는 *인라인 가로 막대 차트* 가 그려져 부서 간 활용 차이를 한눈에 비교할 수 있습니다.

### 6-4. 위젯 드릴다운 링크 제거

거버넌스·시스템 그룹 위젯에서 *드릴다운 링크* (위젯 클릭 → 관리 페이지 이동) 가 제거되었습니다. 사용자는 사이드바 메뉴를 통해 직접 진입해야 합니다.

## 7. 지원 페이지 딥링크

대시보드 위젯에서 *공지사항·자주 묻는 질문·관리자 문의* 항목을 클릭하면 해당 게시물이 자동으로 펼쳐진 상태로 페이지에 진입합니다.

| 위젯 항목 | 동작 |
|---|---|
| 자주 묻는 질문 (FAQ) | `?faqId=` 쿼리로 진입 → 해당 질문이 자동 펼침 |
| 공지사항 | `?noticeId=` 쿼리로 진입 → 해당 공지로 스크롤·강조 |
| 관리자 문의 | `?qnaId=` 쿼리로 진입 → 해당 문의 상세 모달 자동 오픈 |

## 8. PII 사전 스캔 — 스트리밍 진행 표시

지식관리 *문서 업로드* 시 PII 사전 스캔이 **SSE (Server-Sent Events)** 방식으로 변경됐습니다.

| 항목 | 이전 | 이후 |
|---|---|---|
| 통신 방식 | 단일 POST → 결과 한 번에 반환 | SSE 스트림 + 30초 하트비트 |
| 최대 처리 시간 | 120 초 | 600 초 (10 분) |
| 진행 표시 | 단일 진행률 막대 | 실시간 단계 메시지 |

**효과**: 대용량 문서의 PII 스캔 도중 타임아웃이 발생하지 않으며, 사용자는 처리 단계를 실시간으로 확인할 수 있습니다.

## 9. 알림 센터 — 이벤트 기반 갱신

알림 개수 갱신 전략이 *5초 간격 폴링* 에서 **이벤트 기반** 으로 변경됐습니다.

| 트리거 | 동작 |
|---|---|
| 페이지 라우트 변경 | 즉시 갱신 |
| `?view=` 쿼리 변경 | 즉시 갱신 |
| 창 포커스 복귀 | 즉시 갱신 |
| 모달 열기 / 가시성 변경 | 즉시 갱신 |

**효과**: 네트워크 호출 빈도가 감소하고, 사용자가 화면을 전환할 때 더 빠르게 알림이 동기화됩니다.

## 10. 컬렉션 사전(Dictionary) 관리

지식 컬렉션에 *사전(Dictionary)* 단위 CRUD 가 추가됐습니다. 동의어·금칙어·도메인 용어 묶음을 사전으로 등록해 컬렉션 검색에 활용할 수 있습니다.

| 동작 | 경로 |
|---|---|
| 사전 목록 조회 | 컬렉션 상세 → **사전** 탭 |
| 사전 생성 | 사전 탭 우상단 **+ 사전 추가** |
| 용어 일괄 등록 | 사전 상세 → **벌크 가져오기** (CSV 업로드, 네이티브 fetch + FormData) |
| 컬렉션 적용 | 컬렉션 생성·편집 모달 → *사전 선택* 드롭다운 |

## 11. 온톨로지 빌드 — 취소 후 404 정상 처리

지식 컬렉션 온톨로지 빌드 중 *취소* 후 폴링이 404 를 받는 경우, **빌드 종료**로 정상 처리됩니다 (이전: 무한 폴링).

| 시나리오 | 동작 |
|---|---|
| 빌드 진행 중 취소 클릭 | 즉시 취소 요청 전송 |
| 취소 후 상태 폴링 | 404 응답 → 빌드 종료로 처리, 폴링 중단 |

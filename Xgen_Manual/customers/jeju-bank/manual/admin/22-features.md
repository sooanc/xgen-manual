# 관리자 기능 가이드

본 챕터는 **관리 설정** 모드의 주요 기능을 영역별로 정리합니다. 각 항목은 화면 진입 경로와 함께 운영 절차를 안내합니다.

!!! info "스크린샷 보강 예정"
    본 챕터의 일부 절은 다음 회차 캡처에서 스크린샷을 추가합니다. 현재는 텍스트 기반 화면 설명·라벨만 정리되어 있습니다.

## AI 모델 관리 · 모델 서빙 멀티노드

여러 GPU 노드(서버)를 운용하는 환경에서 어느 서버에 모델을 로드·실행할지 직접 지정할 수 있습니다.

**진입 경로**

좌측 사이드바 **관리 설정 → AI 모델관리 → 모델 서빙 → 모델 로드 모달**

**모델 로드 절차**

1. 모델 서빙 화면 우상단 **+ 모델 로드** 클릭
2. 모달에서 다음 항목 입력
    - **모델 선택** — 등록된 모델 목록에서 선택
    - **서버 (노드) 선택** — 모델을 적재할 서버 노드 드롭다운에서 선택
    - **Parser** — datalist 에서 응답 파서 선택 (자동 완성 지원)
    - **vLLM launch 인자** — 필요 시 추가 실행 인자 입력
3. **로드 실행** 클릭 → 선택한 서버 노드로 라우팅되어 모델이 적재됨

**컨텍스트 OOM 경고**

모달 내부에 *컨텍스트 OOM (메모리 초과)* 발생 가능성을 안내하는 문구가 노출됩니다. 동일 모델을 여러 노드에 중복 로드할 경우 GPU 메모리 점유가 누적되므로 노드 분담 배치를 권장합니다.

## 관리자 대시보드 · 신규 위젯

대시보드 진입 직후 다음 위젯을 통해 운영 현황을 한눈에 확인합니다.

### Agent 현황

배포 승인 처리 큐를 3-stat 으로 즉시 표시합니다. `admin.governance-review:read` 권한이 있는 사용자에게 노출됩니다.

| stat | 의미 |
|---|---|
| 배포 승인 대기 | 시스템 관리자 1차 승인 대기 건수 |
| 승인 반려 (완료 후) | 거버넌스 단계에서 반려된 건수 |
| 승인 반려 | 시스템 관리자 1차 단계에서 반려된 건수 |

### 점검 기한 초과 현황

거버넌스 점검 일정을 4-stat 으로 노출합니다.

| stat | 의미 |
|---|---|
| 점검 기한 초과 | 점검 일정이 지났으나 미완료된 에이전트 건수 |
| 점검 예정 | 향후 점검 일정이 등록된 건수 |
| 점검 완료한 | 최근 점검을 완료한 건수 |
| 전체 대상 에이전트 | 점검 대상 에이전트 총수 |

### 위험 평가 재수행 대상

거버넌스 승인 흐름을 4-stat 으로 노출합니다.

| stat | 의미 |
|---|---|
| 신규 배포승인 완료 | 배포 승인이 갱신된 건수 |
| AI 에이전트 미승인 건수 | 거버넌스 미승인 상태 건수 |
| 미승인 비율 (%) | 미승인 / 전체 비율 |
| 승인 비율 (%) | 승인 / 전체 비율 |

### 부서별 현황 (테이블 시각화)

부서별 활용 현황을 4-컬럼 테이블로 비교합니다.

| 컬럼 | 내용 |
|---|---|
| 부서명 | 부서 이름 |
| 활성 Agent | 부서별 활성 에이전트 수 + 가로 막대 차트 (현재 표시 행 중 최대값 기준) |
| 실행수 | 누적 실행 횟수 (정렬 기준) |
| 사용자 | 활성 사용자 수 |

활성 Agent 컬럼의 인라인 막대를 통해 부서 간 활용 차이를 즉시 비교할 수 있습니다.

## 관리자 대시보드 · 사이드바 빠른 이동

대시보드 환영 카드 우측에 **Teams** · **Canvas** 빠른 이동 버튼이 노출됩니다. 권한이 있는 관리자에게만 활성 상태로 표시됩니다.

| 버튼 | 이동 경로 | 필요 권한 |
|---|---|---|
| Teams | `/teams` | (없음) — 항상 활성 |
| Canvas | `/canvas` | `main.agentflow-canvas:read` 또는 `:manage` |

## 알림 센터

사이드바 헤더의 알림 아이콘 옆 카운트는 다음 시점에 즉시 갱신됩니다.

- 페이지 이동
- 사이드바 메뉴 전환
- 브라우저 창 포커스 복귀
- 모달 열기·닫기

별도 새로고침 없이 최신 운영 알림을 즉시 확인할 수 있습니다.

## 운영 권장사항

- **모델 서빙 노드 분산** — 동일 모델을 여러 노드에 중복 로드하지 말고 사용 패턴에 따라 분담 배치합니다.
- **점검 기한 초과 모니터링** — 대시보드 *점검 기한 초과 현황* 카운트가 일정 건수를 넘으면 거버넌스 점검 일정을 재조정합니다.
- **부서별 활성도 검토** — *부서별 현황* 표를 월 1회 검토해 비활성 부서·과부하 부서를 식별합니다.

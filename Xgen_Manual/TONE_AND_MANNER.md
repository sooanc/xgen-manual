# AI 솔루션 가이드 작성 톤 & 매너 가이드

본 파일은 **"솔루션 가이드 자동화" 스크립트** 가 AI 로 매뉴얼 본문을 생성·재작성할 때 따라야 할 톤·매너 규칙입니다. 사람이 직접 [base/**/*.md](base) 본문을 편집할 때도 동일하게 적용합니다.

> 본 파일과 [AUTHORING.md](AUTHORING.md) 는 서로 다른 층위입니다.
> - [AUTHORING.md](AUTHORING.md) — *무엇을* 적을지에 관한 사실 규칙 (용어, UI 사실, 빌드·검증 절차)
> - [TONE_AND_MANNER.md](TONE_AND_MANNER.md) — *어떻게* 적을지에 관한 문체·표현 규칙
>
> 두 파일을 모두 따르되, 사실 정합성(AUTHORING) 이 표현 스타일(TONE) 보다 우선합니다.

## 1. 기본 원칙

솔루션 가이드는 사용자가 기능을 빠르고 정확하게 이해하고 실행할 수 있도록 돕는 문서입니다. AI 를 활용해 문서를 작성하더라도, 일관된 용어와 전문적인 문체를 유지해야 합니다.

문서는 "기술 설명"보다 "사용자 이해"를 우선으로 작성합니다.

## 2. 문체 원칙

### 2-1. 명확하고 간결하게 작성

복잡한 표현보다 짧고 직관적인 문장을 사용합니다.

**권장**

- Agent 를 실행합니다.
- 권한을 설정할 수 있습니다.
- 결과를 저장합니다.

**지양**

- Agent 를 기반으로 실행 처리가 가능하도록 제공됩니다.
- 권한 설정 기능이 지원되고 있습니다.

### 2-2. 설명형 문체 사용

사용 방법과 동작 원리를 차분하게 설명하는 문체를 사용합니다.

**권장**

- 사용자는 Agent 를 생성할 수 있습니다.
- 설정된 권한은 즉시 반영됩니다.
- 승인 후 운영 환경에 배포됩니다.

**지양**

- 지금 바로 Agent 를 만들어보세요!
- 매우 강력한 기능을 제공합니다!

### 2-3. 객관적 표현 유지

과장되거나 마케팅성 표현은 최소화합니다.

**권장**

- 실시간 실행 상태를 확인할 수 있습니다.
- 운영 정책에 따라 접근이 제한될 수 있습니다.

**지양**

- 혁신적인 AI 경험을 제공합니다.
- 압도적인 성능을 체험하세요.

## 3. 용어 사용 기준

### 3-1. 용어는 일관되게 유지

동일한 기능은 항상 동일한 용어를 사용합니다.

| 권장 | 지양 |
|---|---|
| Agent | 봇 / AI / 워커 혼용 |
| 실행 | 동작 / 수행 / 런 혼용 |
| 배포 | Publish / 오픈 혼용 |
| 권한 | 접근권 / 인증권한 혼용 |

> 더 정밀한 용어 통일 규칙(에이전트 vs 에이전트플로우, SuperUser vs Standard User 등)은 [AUTHORING.md](AUTHORING.md) 1절·2.5절을 우선 참고합니다.

### 3-2. 영문 용어 최소화

가능한 한 한글 중심으로 작성합니다.

**권장**

- 실행 결과
- 접근 권한
- 참조 문서

**예외 — 업계 표준 용어로 허용**

- Agent
- Prompt
- API
- Workflow
- Token
- Streaming

## 4. 사용자 중심 작성 원칙

### 4-1. 사용자가 해야 할 행동 중심으로 작성

기능 자체보다 사용자의 행동 흐름 중심으로 설명합니다.

**권장**

1. Agent 를 선택합니다.
2. 입력값을 설정합니다.
3. 실행 버튼을 클릭합니다.

**지양**

- 본 기능은 입력값 설정 기능을 제공합니다.

### 4-2. 결과를 함께 설명

기능 실행 후 어떤 결과가 발생하는지 함께 안내합니다.

**예시**

- 저장 후 즉시 적용됩니다.
- 승인 이후 운영 환경에서 사용할 수 있습니다.
- 실행 결과는 이력 화면에서 확인할 수 있습니다.

## 5. UI/UX 문구 작성 기준

### 5-1. 버튼 문구

동사 중심으로 짧게 작성합니다.

| 권장 | 지양 |
|---|---|
| 저장 | 저장하기 |
| 실행 | 실행 요청 |
| 배포 | 운영 배포 진행 |

### 5-2. 안내 문구

불필요한 감탄 표현을 사용하지 않습니다.

**권장**

- 승인 후 배포할 수 있습니다.
- 권한이 없는 경우 접근이 제한됩니다.

**지양**

- 아주 쉽게 사용할 수 있습니다!
- 지금 바로 시작해보세요!

## 6. AI 문서 생성 시 금지 사항

다음 표현은 사용하지 않습니다.

- 과도한 마케팅 표현
- 추상적인 표현
- 의미 없는 수식어
- 중복 설명
- 불명확한 기술 용어
- 사용자 책임이 불분명한 문장

**예시 (사용 금지)**

- 최고의 경험
- 혁신적인 플랫폼
- 다양한 기능 제공
- 효율적인 업무 환경

## 7. 권장 톤

본 솔루션 가이드는 아래 톤을 유지합니다.

- 전문적
- 명확함
- 차분함
- 신뢰감
- 운영 중심
- 기업 환경 친화적

문서는 **"홍보자료"** 가 아니라 **"운영 가이드"** 라는 기준으로 작성합니다.
